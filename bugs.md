# Bugs

This file tracks concrete, actionable defects. Do not add vague risks, general cleanup ideas, or architecture wishes here.

## Entry Format

- `Title`: short defect label
- `Status`: `open`, `investigating`, `blocked`, or `fixed`
- `Impact`: who or what is affected
- `Reproduction`: the smallest known way to observe the issue
- `Expected`: correct behavior
- `Actual`: current behavior
- `Scope / Suspected Area`: likely files, subsystems, or commands involved
- `Verification`: how to confirm a fix
